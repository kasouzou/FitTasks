# Kotlin 修飾子と関連キーワードの説明

- `package`：ソースが属する名前空間を定義します。ファイル冒頭に１つだけ書かれ、同じパッケージのクラスからパッケージ名なしで参照できるようにします。
- `import`：外部クラスや関数を明示的に取り込んでおくための宣言です。フルパッケージ名の 1 行を短縮してコード内で使えます。
- `class`：Kotlin の設計単位であるクラスを定義するキーワードです。継承する場合は `:` でスーパークラスを指定します。
- `object`：シングルトン（唯一のインスタンス）を表すオブジェクト宣言です。`sealed interface` の内部状態や定数的な振る舞いを定義する際に使われます。
- `data class`：主にデータを保持するクラスで、自動的に `equals` や `copy` などのメソッドを生成します。内部のプロパティ構造が重要な場合に使います。
- `sealed`：指定したインターフェースやクラスは同じファイル内でしかサブクラスを定義できず、when 式で網羅性チェックができるようになります。
- `interface`：実装を持たない契約を表現するための型です。`sealed interface` で使うとサブタイプの範囲を厳密に限定できます。
- `override`：基底クラスやインターフェースのメソッド/プロパティを再定義する際に使う修飾子で、明示的な宣言が必須です。
- `private`：同じクラス内からのみアクセスできるように可視性を制限します。状態のカプセル化に使います。
- `val`：再代入不可の読み取り専用プロパティを定義するキーワードです。イミュータブルな状態を保持するときに使います。
- `var`：再代入可能なプロパティを定義します。UI 状態を更新したい場合に `var` + delegated property と組み合わせて使うことが多いです。
- `fun`：関数を定義するキーワードです。戻り値の型を明示的に書くか、式の形で省略できます。
- `by`：委譲プロパティを宣言するために使います。本コードでは `var foo by remember { mutableStateOf(...) }` のように Compose の状態をプロパティに委譲しています。
- `@Composable`：Compose の描画関数に付けるアノテーションです。これがないと Compose 関数として呼べません。
- `@OptIn`：Experimental API を使う際に必要なアノテーションで、許可する Experimental クラスを明示します。
- `@Suppress`：コンパイラ警告を抑制する注釈です。`"UNCHECKED_CAST"` のように警告名を指定します。
- `remember`：Compose が再コンポジションしても値を保持するための関数です。`mutableStateOf` や `mutableStateListOf` と組み合わせて状態を持たせます。
- `mutableStateOf`, `mutableStateListOf`：Compose の再描画トリガーとなる状態コンテナを生成します。`remember` と一緒に使うことで UI による再描画が適切に発火します。
- `Scaffold`、`Column`、`Row` などの Composable はクラスではなく関数ですが、共通して `modifier` の引数に `Modifier` を渡すことでレイアウトを調整します。
